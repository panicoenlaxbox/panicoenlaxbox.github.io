Medir líneas de código, clases y funciones en Python

 Publicado el December 23, 2023

Lo cierto es que no importa el número, lo relevante es si un código funciona y es mantenible. No obstante, como ejercicio de aprendizaje y para contemplar el código desde otra perspectiva, este tipo de métricas pueden resultar divertidas.

Este post surge a raíz de ver un vídeo en YouTube How Large is Your Project? Count The Lines with cloc! y probar la herramienta cloc. Está bien, pero no me convenció del todo. Sin embargo, me picó la curiosidad e investigué un poco sobre ello.

Yo quería jugar con el módulo ast, Abstract syntax tree y me pareció una oportunidad buena para ponerlo en práctica.

En VSCode hay un plugin muy útil que te ayudará a ver el ast del fichero .py actual de forma muy fácil.

Como guía de supervivencia de ast (un poco desordenada pero suficiente), ahí va este snippet:

import ast
from _ast import FunctionDef, ClassDef

code = """
import threading


class Foo:
    def __init__(self, bar: str) -> None:
        self.bar = bar

    def baz(self) -> None:
        print(self.bar, threading.get_ident())


foo = Foo("bar")
foo.baz()


def qux(quux: str) -> str:
    return quux.upper()


qux("quux")

print("Hello, world!")
    """

tree = ast.parse(code)

compiled_code = compile(source=tree, filename="", mode="exec")
exec(compiled_code)

# bar 11412
# Hello, world!


# For visiting an AST, there are two main approaches: helper functions or with the ast.NodeVisitor class.

print("ast.walk")
for node in ast.walk(tree):
    print(f"\t{node.__class__.__name__}")

print("ast.iter_fields")
for name, value in ast.iter_fields(tree):
    print(f"\t{name} {value}")

print("ast.iter_child_nodes")
for node in ast.iter_child_nodes(tree):
    print(f"\t{node.__class__.__name__}")


# With ast.NodeVisitor, one can register specific callbacks to trigger when visiting specific node types


class MyVisitor(ast.NodeVisitor):
    def visit_ClassDef(self, node: ClassDef) -> None:
        # callback to be triggered when ast.FunctionDef nodes are visited
        print(f"\tfound ClassDef {node.name} at line: {node.lineno}")
        # propagate the visit on the children of the input node
        self.generic_visit(node)

    def visit_FunctionDef(self, node: FunctionDef) -> None:
        # callback to be triggered when ast.FunctionDef nodes are visited
        print(f"\tfound FunctionDef {node.name} at line: {node.lineno}")
        # propagate the visit on the children of the input node
        self.generic_visit(node)


print("MyVisitor")
my_visitor = MyVisitor()
my_visitor.visit(tree)

Habiendo puesto ya en el mapa al módulo ast, ahora toca hacer algo de valor con él y es ahí donde se podría usar para medir símbolos en nuestro código.

Para el número de clases y funciones (yo me he limitado a eso pero es cuestión de ir registrando callbacks para otro tipo de elementos que vayamos visitando) he usado ast. Por el contrario, para contar las líneas de código he usado Python puro y duro con lo que, sería muy mejorable si quisiéramos por ejemplo no contar comentarios de tipo docstring o cualquier otra cosa que se te ocurra. Yo (no sé si por pereza o por convencimiento pleno) he llegado a la conclusión de que los docstring son código, así que no me siento culpable por contarlos.
